# Ressources pour aller plus loin

## Documentation officielle

- **React** — react.dev : la documentation officielle, réécrite en 2023, avec des explications interactives (sandbox intégrés) et un fil directeur pédagogique proche de celui de ce manuel.
- **Vite** — vitejs.dev : guide et référence de configuration complète.
- **React Router** — reactrouter.com : documentation de la version utilisée dans ce manuel.
- **Redux Toolkit** — redux-toolkit.js.org : le guide officiel "Redux Essentials" approfondit chaque concept du chapitre 21.
- **TypeScript** — typescriptlang.org/docs : le "Handbook" officiel, référence complète du langage.

## Librairies de formulaires et validation

- **React Hook Form** — react-hook-form.com : documentation avec exemples interactifs pour chaque hook (`useForm`, `useFieldArray`, `Controller`).
- **Zod** — zod.dev : documentation complète du système de schéma.
- **Yup** — github.com/jquense/yup : documentation historique, toujours maintenue.

## Librairies UI

- **Tailwind CSS** — tailwindcss.com/docs : référence exhaustive de toutes les classes utilitaires.
- **Material UI** — mui.com : documentation avec un configurateur de thème visuel interactif.
- **Ant Design** — ant.design : documentation avec de nombreux exemples de composants prêts à copier.
- **React-Bootstrap** — react-bootstrap.github.io : documentation des composants.

## Tests

- **Vitest** — vitest.dev : documentation, très proche de celle de Jest.
- **React Testing Library** — testing-library.com/docs/react-testing-library/intro : inclut le guide de bonnes pratiques ("Common mistakes").
- **Playwright** — playwright.dev : pour les tests end-to-end en navigateur réel, déjà utilisé dans plusieurs projets de Jaslin (BANKA, GESCOM, EduSpher).

## Performance et outils de développement

- **React DevTools** — extension navigateur officielle (Chrome, Firefox, Edge), indispensable pour le Profiler (chapitre 40).
- **web.dev/vitals** — référence des métriques de performance web (Core Web Vitals) mesurées par Lighthouse.

## SEO et rendu serveur

- **Next.js** — nextjs.org/docs : pour approfondir le SSR/SSG évoqué au chapitre 45.

## Déploiement

- **Vercel** — vercel.com/docs
- **Netlify** — docs.netlify.com

## Accessibilité

- **WAI-ARIA Authoring Practices** — w3.org/WAI/ARIA/apg : catalogue de patterns d'accessibilité par type de composant (menu, modale, onglets...).
- **WebAIM Contrast Checker** — webaim.org/resources/contrastchecker : vérifier le ratio de contraste d'une palette de couleurs.

## Communautés et veille

- **Reddit r/reactjs** — discussions et retours d'expérience de la communauté.
- **Bluesky/X, comptes de l'équipe React officielle** — annonces de nouvelles fonctionnalités en premier lieu.
- **State of JS** (état de l'art annuel) — stateofjs.com : tendances de l'écosystème JavaScript/React, utile pour suivre l'évolution des choix de librairies (Redux vs Zustand, Yup vs Zod...) au fil du temps.
