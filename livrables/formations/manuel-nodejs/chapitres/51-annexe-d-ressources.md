# Ressources pour aller plus loin

## Documentation officielle

- **Node.js** — nodejs.org/docs : documentation officielle de l'API native (fs, http, path, worker_threads...).
- **Express.js** — expressjs.com : guide officiel, référence complète de l'API.
- **npm** — docs.npmjs.com

## Bases de données et ORM

- **PostgreSQL** — postgresql.org/docs
- **MongoDB** — mongodb.com/docs
- **Prisma** — prisma.io/docs : documentation particulièrement soignée, avec un guide interactif.
- **Sequelize** — sequelize.org/docs
- **Mongoose** — mongoosejs.com/docs

## Sécurité

- **OWASP Top 10** — owasp.org/www-project-top-ten : référence des vulnérabilités web les plus critiques, essentielle pour tout développeur backend.
- **Helmet** — helmetjs.github.io
- **jsonwebtoken** — github.com/auth0/node-jsonwebtoken

## Tests

- **Jest** — jestjs.io/docs
- **Supertest** — github.com/ladjs/supertest

## Documentation d'API

- **OpenAPI Specification** — spec.openapis.org
- **Swagger UI** — swagger.io/tools/swagger-ui

## Conteneurisation et déploiement

- **Docker** — docs.docker.com
- **Docker Compose** — docs.docker.com/compose
- **Railway** — docs.railway.app (déploiement simplifié, bases de données managées)

## Performance et supervision

- **PM2** — pm2.keymetrics.io/docs
- **Winston** — github.com/winstonjs/winston

## Écosystème plus large (pour approfondir après ce manuel)

- **TypeScript avec Node.js** — typescriptlang.org/docs : pour un typage statique complet, en complément du JavaScript pur de ce manuel.
- **NestJS** — nestjs.com : framework Node.js plus structuré qu'Express, inspiré d'Angular, pour des projets à grande échelle.
- **GraphQL** — graphql.org : alternative à REST pour certains besoins d'API, notamment quand le client doit contrôler précisément les champs retournés.

## Communautés et veille

- **Node.js Weekly** (newsletter) — nodeweekly.com
- **Stack Overflow** (tags `node.js`, `express`, `prisma`)
- **r/node** (Reddit)
